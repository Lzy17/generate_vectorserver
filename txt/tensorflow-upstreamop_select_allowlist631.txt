<p>The following snippet shows how to convert models with the above operators:</p>
<p>```python
import tensorflow as tf</p>
<h1>These imports are required to load operators' definition.</h1>
<p>import tensorflow_text as tf_text
import sentencepiece as spm</p>
<p>converter = tf.lite.TFLiteConverter.from_keras_model(your_model)

  tf.lite.OpsSet.TFLITE_BUILTINS, tf.lite.OpsSet.SELECT_TF_OPS

```</p>
<p>On the runtime side, it is also required to link the TensorFlow Text or
SentencePiece library into the final app or binary.</p>
<h2>User's defined Operators</h2>
<p><em>Note: This feature is only available from TensorFlow 2.5 version</em></p>
<p>If you
<a href="https://www.tensorflow.org/guide/create_op">created your own TensorFlow operators</a>,
you can also convert models containing them to TensorFlow Lite by listing
required operators in the <code>experimental_select_user_tf_ops</code> as following:</p>
<p>```python
import tensorflow as tf</p>
<p>ops_module = tf.load_op_library('./your_ops_library.so')</p>
<p>converter = tf.lite.TFLiteConverter.from_saved_model(your_model)
converter.target_spec.supported_ops = [
  tf.lite.OpsSet.TFLITE_BUILTINS, tf.lite.OpsSet.SELECT_TF_OPS
]
converter.target_spec.experimental_select_user_tf_ops = [
    'your_op_name1',
    'your_op_name2'
]
model_data = converter.convert()
```</p>
<p>On the runtime side, it is also required to link your operators library into the
final app or binary.</p>
<h2>Add TensorFlow core operators to the allowed list.</h2>
<p>If you hit the case where the TensorFlow core operators are not in the above
allowed
<a href="https://www.tensorflow.org/lite/guide/op_select_allowlist#tensorflow_core_operators">list</a>,
you can report the feature request at
<a href="https://github.com/tensorflow/tensorflow/issues">here</a> with the names of the
TensorFlow core operators, not listed in the allowed list.</p>
<p>You can also create own your pull request from the source code. For example, if
you want to add the <code>raw_ops.StringToNumber</code> op in the allowed list, there are
three places to update like this
<a href="https://github.com/tensorflow/tensorflow/commit/02e691329517eb5e76522ed8d8bef79ceb082ff8">commit</a>.</p>
<p>(1) Add the operator kernel source code to the <code>portable_extended_ops_group2</code>
BUILD rule.</p>
<p>```
filegroup(
    name = "portable_extended_ops_group2",
    srcs = [
        ...
+       "string_to_number_op.cc",</p>
<pre><code>    ...
],
</code></pre>
<p>)
```</p>
<p>In order to find the relevant operator kernel source file under the
<code>tensorflow/core/kernels</code> directory, you can search the source code location,
which contains the following kernel declaration with the operator name:</p>
<p><code>REGISTER_KERNEL_BUILDER(Name("StringToNumber")                 \
                            .Device(DEVICE_CPU)                \
                            .TypeConstraint&lt;type&gt;("out_type"), \
                        StringToNumberOp&lt;type&gt;)</code></p>
<p>If there are any header files under the <code>tensorflow/core/kernels</code> directory,
required in the operator kernel source code, you need to add the header file
into the <code>portable_extended_ops_headers</code> BUILD rule as the follows:</p>
<p>```
