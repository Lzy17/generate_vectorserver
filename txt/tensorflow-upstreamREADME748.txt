<h1>Experimental C++ Ops</h1>
<p>The C++ files in this directory (<strong><em>*_ops.h</em></strong> and <strong><em>*_ops.cc</em></strong>) are
autogenerated from the registered Op and API definitions.</p>
<p>To regenerate them, run the script in this directory, <code>update_cpp_ops.sh</code>, with
no arguments. This script will overwrite the existing ops in-place at
<strong><em>tensorflow/c/experimental/ops/*_ops.{cc,h}</em></strong>.</p>
<p>Run this <code>update_cpp_ops.sh</code> script when Op definitions change in the registry.</p>
<p>To generate additional operators, extend the lists in this script. Note that
category names correspond to generated source file names, and should be
consistent with the original source files registering each operator. For example
since <code>REGISTER_OP("MatMul")</code> appears in <strong><em>core/math_ops.cc</em></strong>, the "MatMul"
operator in the script should be in the "math" category, and it will be
generated in the output file <code>c/experimental/ops/math_ops.cc</code>.</p>
<p>Running this script should be a no-op, generating identical code other than
formatting (i.e., line wrapping).</p>
